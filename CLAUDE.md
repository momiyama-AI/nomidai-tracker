# Claude Code 開発規約

このファイルはClaude Code向けの作業規約である。実装前に必ず `SPEC.md` と本ファイルを読むこと。

## 最重要ルール

- `SPEC.md` が単一の真実。迷ったら `SPEC.md` に従う。
- 仕様が曖昧な場合は、`SPEC.md` 末尾の推奨仮決めを採用し、必要なら質問を残す。
- フェーズごとにスコープを守る。先のフェーズの機能を作り込まない。
- 各フェーズ終了時にビルド確認とcommitを行う。
- コミットメッセージは日本語1行にする。
- SwiftDataマイグレーションを壊す変更は、実装前に必ず人間へ事前申告する。

## 実装方針

- Swift 5.10+ / SwiftUI / iOS 17.0+。
- SwiftDataを使う。
- アーキテクチャはMV + Repository層。過剰な抽象化、巨大なDIコンテナは禁止。
- 外部ライブラリは原則追加しない。
- サーバー、外部API、アカウント登録を追加しない。
- 金額は必ず `Int` 円で扱う。金額に `Double` / `Decimal` を使わない。
- 度数と純アルコール量も、`SPEC.md` の整数計算方針に従う。
- 日付/月境界は `Calendar.current` を使い、ユーザーのタイムゾーン基準にする。
- 文字列は `Localizable.strings` に集約する。
- 全画面に `#Preview` を用意する。

## App Store審査セーフティ

- アルコールを扱うため年齢制限17+前提。
- 健康、医療、節酒、禁酒、依存症改善などの効果を謳わない。
- 純アルコール量は参考情報としてだけ扱う。
- マスコットのセリフで説教、健康注意、飲酒量への善悪評価をしない。
- StoreKit実装時は購入復元、利用規約リンク、プライバシーポリシーリンクを必ず置く。
- Privacy Manifestはデータ収集ゼロとして整備する。

## フェーズ運用

各フェーズで以下を実施する。

1. `SPEC.md` とこのファイルを読む。
2. 指示されたフェーズ範囲だけ実装する。
3. 関連テストを追加する。
4. `xcodebuild` などでビルド/テスト確認する。
5. 結果を短くまとめる。
6. 日本語1行でcommitする。

## 禁止事項

- `SPEC.md` にない外部API、サーバー、ログイン機能を追加すること。
- 健康改善や節酒効果を示唆する文言を入れること。
- 金額計算に浮動小数点を使うこと。
- SwiftDataモデルの保存項目を無断で削除/リネームすること。
- 他フェーズの有料機能やWidgetを先行実装すること。
- ユーザーの未コミット変更を勝手に戻すこと。
