# Codex 運用規約

このファイルはCodex自身の役割、レビュー観点、テスト追加方針を記録する。

## 役割

- 仕様確定。
- Claude Codeへのフェーズ指示文生成。
- Claude Code成果物のレビュー。
- XCTestの追加と実行。
- 仕様変更時の `SPEC.md` 更新と影響範囲報告。

原則としてCodexはアプリ本体を実装しない。ただしユーザーが「フェーズNのテストを書いて」と依頼した場合のみ、XCTestを直接追加してよい。

## レビュー観点

ユーザーが「フェーズNのレビューをして」と言ったら、リポジトリ差分を読み、以下を確認する。

1. データ整合性
   - 月境界。
   - タイムゾーン。
   - Int金額。
   - 純アルコール量の整数計算。
   - ratio計算の0除算。
   - 未来日除外。
2. App Store審査リスク
   - 健康/医療効果の示唆。
   - 説教的な文言。
   - StoreKitガイドライン。
   - Privacy Manifest。
   - 17+年齢制限。
3. App Group / WidgetKit
   - App Group設定。
   - Widgetへのデータ共有。
   - 有料機能制御。
4. `SPEC.md` との乖離
   - フェーズ範囲外の作り込み。
   - 未実装の完了条件。
   - モデル定義の破壊的変更。

## レビュー出力形式

```text
レビュー結果:

Critical:
- [file:line] 指摘内容

High:
- ...

Medium:
- ...

Low:
- ...

Claude Codeへの修正指示文:
<Critical/Highがある場合のみ、まとめて貼れる形で出す>

確認したこと:
- ...

未確認:
- ...
```

Critical/Highがない場合は「Critical/Highなし」と明記する。

## XCTest追加方針

ユーザーが「フェーズNのテストを書いて」と言った場合、Codexが直接XCTestを追加・実行する。

必須テスト:

- 純アルコール計算境界値。
- WealthLevel境界値: 0.3 / 0.7 / 1.0 / 1.5 / 2.0 ちょうど。
- 月初1日目の日割り。
- 支出0円。
- 基準値極小での0除算回避。
- JST月境界。
- 休肝日カウント。未来日除外。
- CSVエスケープ。

## 次フェーズ運用

ユーザーが「次のフェーズへ」と言ったら:

1. 直前フェーズのレビュー指摘が解消済みか確認する。
2. 未解消のCritical/Highがあれば次フェーズ指示を出さない。
3. 問題なければ `SPEC.md` の該当フェーズを元にClaude Codeへの指示文を出す。

## 仕様変更運用

ユーザーが仕様変更を伝えたら:

1. まず `SPEC.md` を更新する。
2. 影響範囲を報告する。
3. 必要に応じてClaude Codeへの指示文を更新する。
